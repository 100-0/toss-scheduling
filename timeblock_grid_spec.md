# 타임블록 그리드 인터랙션 최소 스펙

> 이미지 기반 추정 스펙입니다. 별표(★) 표시된 항목은 이미지만으로 판단이 안 돼서
> 가정한 부분이니 꼭 확인 후 수정해주세요.

## 1. 관련 화면

| 화면 | 타이틀 | 목적 |
|---|---|---|
| 화면 3 | "참석이 어려운 시간이 있나요?" | **불가 시간** 마킹 (캘린더에서 자동 프리필 + 수동 추가) |
| 화면 4 | "조율 가능한 시간이 있나요?" | **양보 가능 시간** 마킹 (선택 사항, 수동 추가만) |
| 모달 | "빠르게 제외하기" | 화면 3 보조 — 프리셋 청크로 대량 제외 |

두 화면은 별개의 입력 패스지만, 결과적으로 시간대별 상태 하나로 합쳐져야 함
(아래 3. 상태 모델 참고).

## 2. 그리드 데이터 구조

```
columns: date[]        // 회의 기간 내 날짜 중 평일만 노출 (주말 제외)
                        // 캡처에서도 7/18(토), 7/19(일)은 노출 안 됨 — 7/17 다음 바로 7/20으로 이어짐
rows: hourSlot[]        // 시각적 셀 단위는 1시간(라벨도 9~18, 20 정수 시간 표기).
                        // 단, 드래그 시작점이 셀의 위/아래 절반이냐에 따라 30분 단위로도
                        // 선택 가능 (아래 4-1 참고) — 그리드는 1시간 셀로 렌더링하되
                        // 내부적으로 30분 스냅 포인트를 가짐

cell = {
  date: string,
  hour: number,
  state: 'default' | 'excluded' | 'flexible' | 'dayoff',
  source: 'calendar' | 'manual' | 'quickExclude'   // 프리필 vs 수동 구분용
}
```

### 컬럼 특수 상태 — "연차" (★비활성 컬럼)
화면 3 캡처에서 7/17 컬럼 전체가 진한 회색으로 음영 처리되고 상단에 "연차" 라벨.
→ 해당 날짜는 드래그 인터랙션 대상에서 제외(탭/드래그 무시), 자동으로 전체 "excluded" 취급.

## 3. 셀 상태 & 색상 (3단계 + 기본)

**화면 4는 화면 3과 같은 전체 그리드(날짜×시간) 레이아웃이 아님.** 화면 3에서 `excluded`로
마킹한 블록들만 따로 추출해서 **압축된 별도 프레임**으로 보여주는 방식 (전체 그리드를 그대로
복제해서 넘기는 게 아니라 excluded 블록만 모아 재배치).

| state | 의미 | 화면 3 컬러 | 화면 4 컬러 | 수정 가능 |
|---|---|---|---|---|
| `default` | 정보 없음 (기본값) | gray10 | (화면4엔 존재하지 않음 — excluded 블록만 노출) | - |
| `excluded` | 참석 불가 | gray80 | gray80 (화면3에서 승계, 미변경 상태) | 탭하면 제거, 드래그로 범위 재조정 |
| `flexible` | 양보 가능 | (해당 없음) | green00 채움 | 탭하면 제거 |
| `dayoff` | 연차 등 통짜 비활성 | gray40, 인터랙션 불가 | **화면4 프레임에서 아예 제외** (양보 불가능한 일정이라 노출 필요 없음) | 불가 |

### 3-1. 화면 4 압축뷰(Compact View) 배치 로직
- 화면 3에서 `excluded`로 마킹된 블록만 골라서 별도 프레임에 재배치 (`dayoff`로 마킹된 블록은 이 프레임에서 완전히 제외)
- **연속된 시간대**(같은 날짜에서 시간이 이어지는 excluded 블록)는 하나로 이어붙여서 표시
- **연속되지 않는 시간대**(사이에 default 상태 셀이 끼어 있어 끊기는 경우)는 블록 사이에 마진(margin)을 줘서 시각적으로 분리
- 즉 화면 4의 프레임 구조는 "날짜×시간 그리드"가 아니라 "excluded 블록들을 시간순으로 나열한 컴팩트 리스트"에 가까움 — 각 블록은 날짜/시간 라벨을 자체적으로 갖고, 인접한 블록끼리만 물리적으로 붙어 보임
- ★프레임 내 블록 정렬 기준(날짜순인지 컬럼 유지인지), 마진 값(px), 한 화면에 다 안 들어갈 때 스크롤 방향은 별도 확인 필요

## 4. 드래그 인터랙션 (핵심)

> 아래 4-1~4-3은 화면 3(전체 날짜×시간 그리드)에 그대로 적용됨.
> 화면 4(압축뷰)에서는 새 블록을 "추가"하는 개념이 없고, 이미 존재하는 excluded 블록
> 내에서 탭(전체 블록 flexible 토글) 또는 드래그(블록 내부 일부 구간만 flexible로 부분 선택)만
> 가능 — 블록 바깥 영역(default 상태)에는 애초에 표시가 없으므로 상호작용 대상이 아님.

### 4-1. 새 블록 추가

**단일 탭** — 빈 셀(1시간 블록) 탭 → 해당 1시간 전체가 즉시 `excluded`/`flexible`로 토글

**드래그** — 30분 단위 스냅 포함, 가로(여러 날짜) 드래그도 지원
1. `pointerdown` 위치가 셀의 상단 절반이냐 하단 절반이냐로 시작 시간 스냅 포인트 결정
   - 예: 9시 셀의 위쪽 절반에서 시작 → 시작점 `9:00`
   - 9시 셀의 아래쪽 절반에서 시작 → 시작점 `9:30`
2. `pointermove` — 세로(시간)뿐 아니라 **가로(날짜)로도 자유롭게 드래그 가능**
   - 드래그 영역은 시작 셀과 현재 포인터 위치를 대각선으로 하는 **사각형 범위**로 계산
     (예: 월~수 컬럼 × 9:00~10:00 로우를 한 번에 선택 → 월/화/수 각각 9-10시가 `excluded`)
   - 세로축은 위와 동일하게 30분 단위 스냅, 가로축은 컬럼(날짜) 단위로 스냅
3. 드래그 중 **플로팅 툴팁이 손가락/커서를 항상 따라다니며** 현재 범위를 실시간 업데이트
   - 단일 날짜만 선택 중일 때: `9:00-` → `9:30-10:30` 형식
   - 여러 날짜 걸쳐 선택 중일 때: `월~수 9:00-10:00` 처럼 날짜 범위 + 시간 범위 함께 표시
   - 즉 툴팁은 드래그 시작과 동시에 뜨고, 손가락을 떼기 전까지 프레임마다 값 갱신
4. `pointerup` — 그 시점의 사각 범위 내 모든 (날짜, 시간) 조합을 `excluded`(화면3) 또는 `flexible`(화면4)로 일괄 커밋
5. 최소 선택 단위: 30분 × 1개 날짜 (드래그 없이 탭만 하면 1시간 단위로 커밋)

### 4-2. 기존 블록 수정/삭제
- 이미 채워진 블록을 **탭** → 삭제 (전체 블록 제거)
- 이미 채워진 블록의 **가장자리(상/하단, 좌/우 끝)를 드래그** → 범위 리사이즈
  - 상/하단 경계: 시간 범위를 30분 단위로 늘리거나 줄임
  - 좌/우 끝 컬럼 경계: 같은 시간대 기준으로 인접 날짜까지 범위 확장/축소
  - 리사이즈 중에도 4-1과 동일하게 플로팅 툴팁으로 갱신된 범위 실시간 표시

### 4-3. 겹침/충돌 처리
- 새 드래그 사각 범위가 기존 블록과 겹치는 (날짜, 시간) 셀이 있으면 → 해당 셀 기준으로 병합(merge), 날짜별로 독립적인 블록 형태 유지 (한 드래그가 여러 날짜에 걸쳐도 각 날짜 컬럼은 개별 블록으로 저장)
- 그리드 경계(상/하단, 좌/우 끝 컬럼) 넘어가는 드래그는 clamp(그리드 범위 내로 제한)
- `dayoff` 컬럼이 드래그 사각 범위에 포함되면 해당 컬럼만 자동 제외(스킵), 나머지 컬럼은 정상 적용

### 4-4. 화면 4 전용 — 선택 안 해도 진행 가능
- 화면 4는 "선택하지 않고 넘어가도 괜찮아요" 카피 존재 → 빈 상태로 CTA(`추천 시간 보기`) 활성화되어야 함
  (화면 3는 회의 기간 필수 선택이라 CTA 비활성 상태로 시작 — 스크린샷상 흐리게 처리됨)

## 5. "빠르게 제외하기" 모달 (Quick Exclude Bottom Sheet)

- 진입: **화면 3에 진입하자마자 자동으로 뜸** (별도 버튼/제스처 트리거 없음)
- 닫기: 우측 상단 X 버튼 탭, 또는 모달 바깥 영역(dimmed background) 탭 → 선택 없이 닫아도 됨 (필수 아님)
- 구성: 프리셋 청크 6개, **다중 선택(multi-toggle)** 칩
  - 평일 17시 이후 / 평일 18시 이후 / 평일 오전 / 점심 시간 / 출근 직후 / 금요일 17시 이후
- "확인" 탭 → 선택된 프리셋들이 그리드에 `excluded` (`source: 'quickExclude'`)로 일괄 반영, 모달 닫힘
- "전체 취소" → 모달 내 선택된 칩 전체 해제 (모달 닫지 않음, ★가정)
- 프리셋 → 실제 시간 매핑

| 프리셋 | 시간 매핑 |
|---|---|
| 평일 17시 이후 | 매 평일 17:00 ~ 그리드 끝 |
| 평일 18시 이후 | 매 평일 18:00 ~ 그리드 끝 |
| 평일 오전 | 매 평일 09:00 ~ 12:00 |
| 점심 시간 | 매 평일 12:00 ~ 13:00 |
| 출근 직후 | 매 평일 09:00 ~ 10:00 |
| 금요일 17시 이후 | 금요일만 17:00 ~ 그리드 끝 |

- 칩 라벨 표기 규칙: 라벨 자체에 시간이 이미 드러나는 칩("평일 17시 이후", "평일 18시 이후", "금요일 17시 이후")은 괄호 생략, 라벨만으론 시간이 안 보이는 칩("평일 오전", "점심 시간", "출근 직후")만 "라벨 (시간)" 형태로 괄호 표기

## 6. 추천 시간 카드 — 3분할(Tri-split) 아바타 배치

### 6-0. 추천 카드 정렬 우선순위
추천 시간 카드(및 시간 확정하기 시트의 비교 테이블에 들어갈 "다른 옵션")는 아래 순서로 정렬:
1. 전원 참석 가능
2. 필수 참석자 전원 가능 (선택 참석자는 무관)
3. 선택 참석자가 양보 가능한 경우
4. 선택 참석자가 불가능한 경우
- 위 우선순위가 동일하게 겹치는 옵션끼리는 **가장 빠른 일정 순**으로 정렬
- 시간 확정하기 시트의 비교 테이블에 들어가는 "우선순위 높은 다른 옵션 2개"는 이 정렬 순서에서 현재 선택한 옵션을 제외한 상위 2개를 그대로 사용

### 6-1. 도형 구조
- 카드는 **정중앙 한 점을 기준으로 3분할**되는 고정 비율의 삼각형/쐐기 형태
  - Zone A (참석 가능) — 좌측 상단 삼각형, 밝은 배경
  - Zone B (참석 불가능) — 우측 상단 삼각형, 어두운 배경
  - Zone C (양보 가능) — 하단 쐐기, **항상 카드 하단 고정 위치**
- 3분할 비율은 **인원 구성과 무관하게 항상 고정**(0명이 배정되는 zone이 있어도 도형 비율은 변하지 않음)
- 분할선은 카드 정중앙 한 점에서 세 방향으로 뻗어나가는 구조로 구현 (예: `clip-path: polygon(...)` 3세트, 중심 좌표 공유)

### 6-2. 아바타 배치 로직
- 참석자별 해당 시간대 상태(가능/불가능/양보가능)에 따라 대응하는 zone에 아바타 배정
- 배치 방향: **각 zone 안에서 위에서 아래로 "떨어지는" 느낌** (zone 상단부터 채워짐, 중력 방향으로 쌓이는 배치)
- 한 zone에 배정된 인원이 없으면 해당 zone은 비워둠 (아바타 없이 배경색만 표시)

### 6-3. 아바타 크기 — 인원수에 따른 자동 축소
전체 참석자 수 기준으로 카드가 꽉 찰 정도면 아바타 크기를 단계적으로 축소:

| 조건 | 아바타 크기 |
|---|---|
| 기본 (~6명 이하) | 48px |
| 약 7명 ~ 15명 | 36px |
| 15명 초과 | 28px |

- 임계값: 6명 이하 48px / 7~15명 36px / 16명 이상 28px
- 축소 판단은 **카드 전체 인원이 아니라 zone별 개별 기준** — 예를 들어 Zone A에 7명이 몰리면 Zone A만 36px로 축소되고, Zone B/C는 인원이 적으면 48px 유지

## 7. 확정 플로우 — "이 시간으로 확정하기" 이후 단계

### 7-1. 전체 흐름
```
추천 시간 화면 ("이 시간으로 확정하기" 탭)
   │
   ├─ 양보 인원 0명 → 바로 "시간 확정하기" 바텀시트로
   │
   └─ 양보 인원 1명 이상 → "양보 요청하기" 바텀시트 먼저 노출
                              │
                              ▼
                        "시간 확정하기" 바텀시트
                              │ ("확정 완료하기" 탭)
                              ▼
                        Complete 화면 ("회의 일정 확정 완료")
```

### 7-2. "양보 요청하기" 바텀시트
- 노출 조건: 선택한 시간대에 **양보 가능 인원이 0명 초과**일 때만 (0명이면 이 시트는 건너뜀)
- 양보 가능 인원 목록을 체크박스로 표시, "요청 보내기" 또는 "아뇨, 괜찮아요" 선택 후 다음 단계("시간 확정하기")로 진행
  (둘 중 뭘 눌러도 다음 시트로 넘어감 — 요청 여부만 다르고 플로우 자체는 안 막힘)

### 7-3. "시간 확정하기" 바텀시트
- 상단: 선택한 날짜/시간 + 체크리스트(가장 빠른 시간대 / 전원 참석 가능 / 양보 없음 등, 해당 시간대 특성에 따라 가변)
- 하단: **참석자별 상태 비교 테이블**
  - 컬럼: 현재 선택한 시간대 **+ 우선순위 높은 다른 옵션 2개** (총 3개 컬럼, 정렬 기준은 6-0 참고)
  - 로우: 참석자별, 각 옵션에서의 상태(가능/양보/불가능) 표시
  - 목적: 확정 직전에 "이 시간이 다른 상위 옵션들과 비교했을 때도 괜찮은지" 마지막으로 검토시켜주는 용도
- "확정 완료하기" 탭 → Complete 화면으로 이동

### 7-4. Complete 화면
- "회의 일정 확정 완료" 타이틀, 확정된 시간/참석자 카드, "확정된 회의 일정이 전원에게 발송됐어요" 안내
- "확인" 탭 시 종료, 하단에 "일정 다시 선택하기" 링크로 재진입 가능

## 8. 요약 — Claude Code에 그대로 넘길 때 우선순위

1. 그리드 렌더링 (날짜 컬럼 × 시간 로우, `dayoff` 컬럼 비활성 처리)
2. 세로 드래그로 블록 생성 + 실시간 툴팁
3. 블록 탭 → 삭제
4. 화면 3 / 화면 4를 같은 컴포넌트로 재사용하되 `mode: 'exclude' | 'flexible'` prop으로 색상/커밋 state만 분기
   (단, `flexible` 모드는 화면3에서 넘어온 `excluded` 셀 위에서만 동작 — 빈 셀 대상 아님)
5. 빠르게 제외하기 모달은 별도 컴포넌트, 그리드에 배치 반영하는 함수만 인터페이스로 연결
6. 추천 시간 카드 3분할 도형 + 아바타 자동 축소 로직 (섹션 6 참고)
7. 확정 플로우: 양보요청 시트 → 시간확정 시트(비교 테이블 포함) → Complete 화면 (섹션 7 참고)

## 9. 데모용 시드 데이터 (핵심 플로우 시연용 고정값)

### 9-1. 6인 전체 시드 데이터
전체 플로우를 처음부터 다 조작하지 않고 **핵심 화면만 빠르게 보여주기 위한 고정 초기 상태**:

| 인원 | 역할 | 제약 |
|---|---|---|
| 이가영 (나) | 필수 | 7/15(수) 외근 14:00~17:00 / 7/17(금) 연차(`dayoff`) |
| 윤지은 | 필수 | 매주 월요일 오전 9:00~11:00 외부 미팅 (`excluded`, 고정 불가능) |
| 박은주 | 필수 | 제약 없음 (거의 항상 가능) |
| 정지훈 | 선택 | 매주 목요일 오후 13:00~17:00 외근(`excluded`, 양보 불가) + 매주 수요일 11:00~12:00 개인 선호 회피(`excluded`→`flexible`, 양보 가능) — **다른 인원 대비 불가능한 날이 많은 케이스**로 고정 |
| 한유진 | 선택 | 정지훈과 동일 패턴 (목요일 외근 공유, 수요일 11-12시 회피) |
| 최영주 | 선택 | 월/화 재택근무일 오전 9:00~12:00 회피 선호(`excluded`→`flexible`, 양보 가능), 그 외엔 항상 가능 |

- **화면 3(참석이 어려운 시간) 캘린더 프리필**은 이가영 기준 위 표대로 이미 `excluded`(`source: 'calendar'`)로 채워진 상태에서 시작
- 나머지 5명의 값은 이가영이 화면 3/4을 조작할 때 "다른 참석자 상태"로 참조만 되고, 직접 조작 대상은 아님 (아래 9-2 참고)

### 9-2. 데모 유저 전환 — 웹 프레젠테이션 레이어 (제품 스펙 아님)
모바일 375px 프레임 옆 웹 캔버스 여백에 **유저 전환 버튼**을 둬서, 두 시점을 오갈 수 있게 함:
- **이가영** — 주최자, 필수 참석자. 새 회의 만들기부터 시작하는 **주최자 전체 플로우**(화면1~5 전체)
- **윤지은** — 필수 참석자, **화면 3 → 화면 4까지만 진행하고 종료** (참석자는 주최자가 아니므로 추천 카드/확정 화면 없음). 화면 4 마지막에 "일정 요청 보내기" 형태로 제출하면 윤지은 플로우 끝
  - 윤지은이 제출한 값(9-1 표의 월요일 오전 제약)은 **이가영 쪽 추천 카드 데이터에 실시간 반영되는 것으로 시연** → 윤지은 전환 전/후로 이가영의 추천 시간 카드 화면(화면5) 결과가 달라지는 걸 보여주는 게 이 데모 토글의 핵심 목적
  - **데모 시나리오(고정 순서)**:
    1. 이가영으로 화면1~5 전체 플로우 완료 (추천 카드 확인)
    2. 윤지은으로 전환 → 화면3, 4 제출
    3. 다시 이가영으로 전환 → 화면5(추천 카드)가 바뀌어 있는 상태로 바로 진입

**정지훈은 전환 대상에 포함하지 않음** — 정지훈은 위 9-1 표대로 "제약이 많은 선택 참석자" 데이터로만 고정해서 다른 유저(이가영/윤지은) 화면에서 참조 데이터로만 등장

- 유저 전환 버튼 옆에 **"초기화" 버튼**도 함께 배치 — 새로고침 대신 버튼 클릭으로 즉시 모든 상태를 9-1 초기값으로 되돌림 (발표 중 재시연 대비, 로딩 깜빡임 없이 매끄러운 전환용)
- ★유저 전환 버튼은 실제 앱 UI가 아니라 데모 프레젠테이션용 토글이므로, Claude Code에 작업 지시할 때 "375px 모바일 프레임 자체엔 없는 요소, 웹 캔버스 레이어에서만 존재"라는 점 명시 필요
